Conclusion

Our experience using Materialize and ASP.net

Materialize Experience

Overall, we had a great experience using materialize. It is very simple and easy to use, and it has modern and innovative designs. Materialize is great for UX designers, as the design of all the components is optimal for a great user experience. Although we made a website, Materialize would be the perfect frontend framework to use for a modern mobile app. Thanks to materialize, we were able to use its elegant components with our own creativity to construct a visually aesthetic and user-centric website!

Pros

  • • Minimal, modern design

  • • Grid systems are easy and fun to use

  • • Creative and different components such as the cards.

  • • Provides a library of colours which can be used to create your own appealing colour pallette

Cons

  • • None!

ASP.net Experience

Microsoft is working to make web applications easier to be created and hosted. In addition to that the dotnet core introduces the model-view-controller concept for web apps to have a strong structure. The learning curve is moderately steep during the installation and early phases. However, it gets steep quickly when MVC concepts are introduced. The structuring can be compared to Python’s Django – it forces you to work their way but at the end their way is the best way and it only gets easier with experience. Dotnet core is robust and it has long time support provided by Microsoft and it being open source makes it favorable for great population of developers. This is one of the best frameworks to work with.

Pros

  • • Cross platform – Windows, Linux

  • • Provides excellent support for utilizing asynchronous programming patterns

  • • Faster – Approximately five times faster than NodeJS

  • • One unified programming model for MVC and Web API

  • • Testability – unit testing is easy

  • • Open source – Vast community of developers

  • • Modular – Provides modularity with middleware components

Cons

  • • Learning curve gets very steep – it’s a vast framework and requires advanced understanding of model-view-controller concept and many other tools

  • • Many tools are windows based and most that run on other platforms will has more functionality on windows.
